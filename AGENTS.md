# Repository Agent Instructions

所有品牌名称、英文名、产品简称和模块命名必须遵循：
`docs/product/BRAND_IDENTITY.md`。

## 项目目标

本仓库开发手机端优先的轻量记账应用，核心业务为：

- 个人收入与支出
- 情侣共同账本
- 个人借贷
- 个人或情侣周期记账
- 个人工资、社保、医保、公积金和个税记录
- 个人或情侣攒钱目标
- 明细、统计、通知、设置和最小运营后台

不得自行增加未在需求文档中确认的业务范围。

## 每次任务开始前

必须依次读取：

1. `docs/index.md`
2. `docs/project-memory/CURRENT_STATE.md`
3. `docs/project-memory/DECISIONS.md`
4. `docs/project-memory/KNOWN_ISSUES.md`
5. 当前任务相关的产品、设计、架构和质量文档
6. 当前 ExecPlan
7. `git status`
8. 未提交的 `git diff`

先输出预检摘要，再修改代码。摘要包括：

- 已读取文件
- 需求理解
- 受影响模块
- 明确不修改的范围
- 风险
- 验证命令

跨模块、数据库、权限、任务队列、重大重构或高风险任务必须先创建或更新 ExecPlan。

## 项目事实来源

优先级：

1. 已批准的业务规则与验收标准
2. 已批准的架构和权限决策
3. 当前 API 与数据库契约
4. 当前状态和最新变更记录
5. 代码和测试

文档与代码冲突时不得猜测。先判断文档是否过期，并在同一任务中同步修复受影响文档。

## 修改原则

- 只修改完成当前任务所必需的文件。
- 不得顺手重构无关模块。
- 不得未经批准增加生产依赖。
- 不得删除、跳过或弱化测试。
- 不得用假数据、静态返回或 TODO 伪装完成。
- 不得覆盖用户未提交修改。
- 不得把敏感信息写入代码、日志、测试数据或文档。
- 修改业务行为时必须引用对应业务规则编号。
- 没有规则时，先补充规则和验收条件，再实现。

## 数据与隐私

- 金额使用整数分，禁止浮点数保存财务金额。
- 业务日期使用 `DATE`；系统时间使用 UTC。
- 跨表财务写操作使用数据库事务。
- 自动生成账目具备数据库级幂等约束。
- 数据读取必须校验用户、账本成员和记录归属。
- 工资、社保、医保、公积金、个人借贷仅本人可见。
- 日志不得包含完整 Token、OAuth 密钥和敏感财务明细。

## 前端

- 手机端优先，遵循 `docs/design/UI_DESIGN_GUIDE.md`。
- Ant Design Vue 作为基础组件，不得让用户端呈现管理后台风格。
- 主题通过设计 Token 实现，禁止散落写死日间/夜间颜色。
- 页面必须覆盖加载、空数据、错误、正常和无权限状态。
- 可点击区域不得小于 44px。
- 危险操作必须二次确认。
- 移动端禁止直接使用宽表格。

## 验证

根据影响范围运行：

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm test:e2e
pnpm build
```

数据库变化还必须运行：

```bash
pnpm prisma validate
pnpm prisma migrate dev
pnpm test:e2e
```

页面变化检查：

- 320px、375px、480px
- 日间和夜间主题
- 空数据、长文本、网络失败
- 快速重复点击和重复提交
- 浏览器返回和刷新恢复

失败必须修复，不能只写入报告。

## 完成后

必须更新：

- `docs/project-memory/CURRENT_STATE.md`
- `docs/project-memory/CHANGELOG_AI.md`
- `docs/project-memory/KNOWN_ISSUES.md`
- `docs/project-memory/DECISIONS.md`（仅产生新决策时）
- 当前 ExecPlan
- 被代码变化影响的需求、API、数据库和设计文档

最终报告说明：

- 完成内容
- 修改文件
- 数据库与 API 变化
- 测试结果
- 未完成内容
- 已知风险
- 下一项建议任务

## Git

- 一个任务对应一个分支或清晰提交序列。
- 提交信息包含任务编号。
- 不提交 `.env`、密钥、生产数据、构建缓存和本地数据库。
- 不使用强制重置删除现有修改。
- 开始和结束时检查 `git status`。

## 完成定义

只有同时满足以下条件才可声明完成：

- 需求和验收标准通过
- 权限与隐私边界正确
- 数据迁移完整
- 幂等和事务约束存在
- lint、类型检查、测试和构建通过
- 文档与项目记忆已更新
- 没有隐藏失败或伪造完成状态
