# 测试策略

## TASK-000 工程基线

- 共享金额/日期校验、配置加载和 UI Token 使用 Vitest。
- 手机端和后台使用 Vue Test Utils 验证最小壳渲染；手机端验证日夜主题切换。
- API 使用 Supertest 验证 `/health`、requestId 透传和统一 404 错误。
- `scripts/test-db-constraints.sql` 验证部分唯一索引、金额上限、来源唯一、跨用户归属和软删除唯一性。
- `scripts/test-migrations.sh` 在一次性 PostgreSQL 中回放迁移，使用独立 shadow 数据库验证零 diff，
  introspection 校验模型数量，并用两个并发连接验证恰好一条写入成功、另一条命中指定幂等唯一索引。
- Chrome DevTools Protocol 验证 320px、375px、480px 无横向溢出和 44px 点击区；业务页面状态留待对应任务。

## API E2E

- 用户和账本
- 情侣邀请并发加入
- 账目越权
- 借贷重复处理
- 周期重复执行
- 周期按用户时区跨 UTC 日期扫描、延迟旧任务跳过、确认通知接收人与数据库去重
- 工资重复到账
- 工资年度 12 月补零、专项代码累计、最近到账周期、PERSONAL 账本限定和固定/日常支出拆分
- 目标并发存入
- 通知私有性
- 导出权限

GitHub `quality` 作业必须同时提供隔离 PostgreSQL 与 Redis，先执行全部生产迁移再运行 `pnpm test:e2e`；不得
依赖无 `siyu_test` 时仅验证 `/health` 的保护性降级冒充完整 E2E。`database` 作业继续独立回放数据库约束脚本。

## 全链路

- 登录 -> 记账 -> 统计
- 创建情侣账本 -> 邀请 -> 双方记账
- 借贷 -> 部分还款 -> 结清
- 周期到期 -> Worker -> 入账
- 工资录入 -> 到账 -> 收入 -> 工资余额
- 情侣攒钱 -> 双方贡献 -> 完成

## 必测边界

- 1 分和大金额
- 月末、闰年、跨年、31 日发薪
- 重复点击、网络重试、并发请求
- 已删除、已退出、已解散、已暂停
- 过期邀请、已满账本
- 逾期和无到期日
- 夜间主题和长昵称/备注
