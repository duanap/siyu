# 业务规则

规则是实现和验收的最高业务事实来源。修改规则必须记录 ADR 和变更原因。

## 用户与账本

- **BR-USER-001**：首次成功登录必须在同一事务中创建用户和唯一个人账本。
- **BR-USER-002**：被禁用用户不得创建或修改业务数据。
- **BR-LEDGER-001**：每个用户有且只有一个有效个人账本。
- **BR-LEDGER-002**：个人账本只能有一个有效成员，成员必须是所有者本人。
- **BR-COUPLE-001**：情侣账本最多两名有效成员。
- **BR-COUPLE-002**：一个用户同一时间最多加入一个有效情侣账本。
- **BR-COUPLE-003**：创建者是情侣账本所有者，另一方是成员。
- **BR-COUPLE-004**：邀请必须有有效期、状态和唯一随机凭证。
- **BR-COUPLE-005**：邀请创建者不能接受自己的邀请。
- **BR-COUPLE-006**：第二名成员加入和邀请状态更新必须在同一事务中完成。
- **BR-COUPLE-007**：解散后双方立即失去访问权限。
- **BR-COUPLE-008**：普通成员只能修改或删除自己创建的共同账目；所有者可管理全部共同账目。
- **BR-COUPLE-009**：工资和个人借贷不得通过情侣账本接口返回。

## 金额、日期与删除

- **BR-MONEY-001**：数据库和内部 API 金额使用整数分。
- **BR-MONEY-002**：金额必须大于零，除非明确业务规则允许冲正。
- **BR-MONEY-003**：展示层负责分与元转换，禁止用浮点数累计财务金额。
- **BR-DATE-001**：业务发生日使用 `DATE`，系统时间使用 UTC 时间戳。
- **BR-DATE-002**：统计日期按用户时区解释。
- **BR-DELETE-001**：财务业务数据默认软删除。
- **BR-DELETE-002**：软删除记录不得进入正常查询和统计。
- **BR-DELETE-003**：删除来源业务记录必须处理其关联账目并写审计日志。

## 普通账目与分类

- **BR-ENTRY-001**：账目必须归属于用户有权访问的有效账本。
- **BR-ENTRY-002**：账目类型只能是收入或支出。
- **BR-ENTRY-003**：账目创建人必须是目标账本有效成员。
- **BR-ENTRY-004**：自动生成账目使用 `source_type + source_id` 唯一约束。
- **BR-ENTRY-005**：普通手工账目 `source_type` 为 `MANUAL`。
- **BR-ENTRY-006**：删除账目后相关统计缓存必须失效。
- **BR-CATEGORY-001**：分类类型必须与账目类型一致。
- **BR-CATEGORY-002**：已被使用的分类只能停用，不能物理删除。
- **BR-CATEGORY-003**：系统分类不可被普通用户改名或删除。

## 借贷

- **BR-DEBT-001**：借到的钱只增加负债，不自动算作普通收入。
- **BR-DEBT-002**：借给别人的钱只增加待收款，不自动算作日常消费支出。
- **BR-DEBT-003**：单次还款或收款金额不得超过剩余金额。
- **BR-DEBT-004**：新增处理记录、更新汇总和可选生成账目必须在同一事务中完成。
- **BR-DEBT-005**：剩余金额为零时自动标记已结清。
- **BR-DEBT-006**：到期日早于当前业务日期且未结清时标记已逾期。
- **BR-DEBT-007**：同一还款记录最多关联一笔普通账目。
- **BR-DEBT-008**：个人借贷仅本人可访问。

## 周期记账

- **BR-RECUR-001**：每个周期规则、每个计划日期最多一个执行实例。
- **BR-RECUR-002**：自动模式在实例生成成功后创建普通账目。
- **BR-RECUR-003**：确认模式先创建待确认实例，用户确认后才创建账目。
- **BR-RECUR-004**：Worker 重试不得重复创建实例或账目。
- **BR-RECUR-005**：达到结束日期或总期数后自动标记完成。
- **BR-RECUR-006**：暂停规则不产生新的到期实例。
- **BR-RECUR-007**：恢复规则重新计算不早于当前日期的下一执行日。
- **BR-RECUR-008**：共同周期规则只能关联情侣账本，双方均可见。
- **BR-RECUR-009**：任务失败记录错误、重试次数和最后执行时间。

## 工资

- **BR-SALARY-001**：同一工资档案和工资月份只能有一条有效记录。
- **BR-SALARY-002**：应发工资等于所有收入项目之和。
- **BR-SALARY-003**：个人扣除等于所有个人扣除项目之和。
- **BR-SALARY-004**：实发工资等于应发工资减个人扣除。
- **BR-SALARY-005**：工资未到账时不得生成普通收入。
- **BR-SALARY-006**：到账生成的普通收入金额必须等于实发工资。
- **BR-SALARY-007**：重复确认到账不得重复生成收入。
- **BR-SALARY-008**：工资、社保、医保、公积金和个税仅本人可见。
- **BR-SALARY-009**：默认工资余额周期从到账日至下一预计发薪日前一天。
- **BR-SALARY-010**：日均可用金额仅在剩余天数大于零时计算。
- **BR-SALARY-011**：年度医保和公积金展示值来自工资记录，不代表官方账户余额。

## 攒钱目标

- **BR-SAVING-001**：目标金额必须大于零。
- **BR-SAVING-002**：目标已存金额等于初始金额加有效存入记录合计。
- **BR-SAVING-003**：存入记录新增、修改或删除与目标汇总更新必须在同一事务中完成。
- **BR-SAVING-004**：已存金额达到或超过目标金额时自动标记完成。
- **BR-SAVING-005**：情侣目标只能关联有效情侣账本。
- **BR-SAVING-006**：情侣贡献按存入记录的用户归属统计。
- **BR-SAVING-007**：完成状态不删除历史存入记录。

## 通知、审计与导出

- **BR-NOTIFY-001**：通知必须归属于明确用户。
- **BR-NOTIFY-002**：用户只能读取和更新自己的通知。
- **BR-AUDIT-001**：删除、解散、权限变化、工资到账和任务手工重试必须写审计日志。
- **BR-EXPORT-001**：导出只能包含当前用户有权访问的数据。
- **BR-EXPORT-002**：导出工资和个人借贷时不得包含情侣成员的私有数据。
