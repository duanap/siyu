本文件是 SIYU 项目品牌名称、简称、模块名称和工程命名的唯一事实来源。

文档版本：V1.0
确认日期：2026-07-11
适用项目：轻量个人与情侣记账应用

# 一、正式品牌名称

## 1.1 中文完整名称

**朝暮同笺 · 四时有余**

固定写法：

```text
朝暮同笺 · 四时有余
```

要求：

- “朝暮同笺”和“四时有余”之间统一使用居中间隔点 `·`
- 间隔点左右各保留一个空格
- 不使用顿号、斜杠、破折号或普通句号替代
- 不得擅自改成“朝暮同笺四时有余”
- 不得擅自拆分为两个互不关联的产品名称

# 三、品牌含义

## 3.1 朝暮同笺

“朝暮”代表每日生活、长久陪伴与时间流转。

“同笺”代表：

- 共同记录
- 共写生活
- 情侣共享账本
- 两个人共同规划日常与未来

品牌表达：

> 朝暮相伴，共书一笺。

## 3.2 四时有余

“四时”代表春夏秋冬、月份周期和生活中的长期积累。

“有余”代表：

- 收支有结余
- 工资有剩余
- 攒钱有积累
- 生活有余裕
- 面对借贷和固定支出时仍有规划

品牌表达：

> 四时有记，生活有余。

## 3.3 完整品牌表达

```text
朝暮同笺 · 四时有余
```

整体含义：

> 记录朝暮生活的每一笔，让四季流转始终有所积累、心中有数、生活有余。

- 应用包名
- 技术文档

- 国际化预留

推荐：

```text

```

该文案只作为英文品牌释义或宣传语，不替代 `SIYU` 作为英文品牌名。

# 五、宣传语

## 5.1 主宣传语

```text
朝暮同笺，四时有余。
```

该句是品牌最核心的中文表达，优先用于：

- 启动页
- 登录页
- 官网首页
- 品牌海报
- 关于应用

## 5.2 辅助宣传语

可选：

```text
记朝暮点滴，留四时有余。
```

```text
每一笔有迹，每一季有余。
```

```text
朝暮皆可记，生活自有余。
```

```text
同记人间烟火，共赴四时有余。
```

## 5.3 使用限制

- 同一页面只保留一句主宣传语
- 不在功能页面频繁堆叠诗意文案
- 业务操作仍应使用清晰直白的文字
- 删除、还款、工资到账、周期扣款等关键操作不得使用含糊古风表达

错误示例：

```text
落笔此账
清偿旧契
银钱已至
```

正确示例：

```text
保存账目
记录还款
确认工资到账
```

品牌文艺感只用于品牌层和情绪层，不影响功能可理解性。

# 十、Codex 执行要求

Codex 在读取本文件后，应遵循以下规则：

1. 将正式产品简称从“轻记账”“Money App”等临时名称统一为“四时有余”。
2. 正式品牌展示场景使用“朝暮同笺 · 四时有余”。
3. 情侣模块默认名称使用“朝暮同笺”。
4. 英文品牌统一使用 `SIYU`。
5. 不得自行翻译或创造新的英文品牌名称。
6. 代码仓库、包名和服务名优先使用 `siyu` 前缀。
7. 不得为了品牌古风感修改业务按钮为含糊文案。
8. 修改品牌相关内容时，必须同步检查：
   - README
   - package.json
   - HTML title
   - PWA manifest
   - 安卓应用名称
   - 管理后台标题
   - 登录页
   - 关于页面
   - 用户协议
   - 隐私政策
   - Docker 服务名
   - 环境变量样例
9. 在正式执行批量重命名前，先搜索所有旧名称并输出影响清单。
10. 批量重命名后必须运行：
    - lint
    - typecheck
    - test
    - build
11. 不得擅自修改数据库业务表名，仅因品牌变化通常不需要重命名数据库表。
12. 不得擅自修改已经发布的 API 路径，除非有单独迁移计划。
13. 本次品牌命名不代表商标、域名、应用商店名称已经完成可用性核验。
14. 正式发布前必须由项目负责人确认商标、域名、软件著作权和应用商店查重结果。
