# TASK-018：工资页面

## 状态

已正式关闭

## 模型与推理

- 推荐模型：Terra
- 推理：Medium
- 是否使用子代理：否

## 目标

- 实现本人私有工资首页 `/salary`，展示当前工资周期余额、当前月工资、年度累计和月度趋势。
- 实现月度工资页 `/salary/:year/:month`，覆盖从档案模板创建、复制紧邻上月、未到账编辑、到账确认和已到账只读。
- 实现年度工资页 `/salary/year/:year`，展示完整 12 月实发趋势、年度应发/实发/月均、奖金和法定扣除累计。
- 提供工资档案首次初始化和后续维护入口，确保创建与到账失败重试复用稳定幂等键。

## 非目标

- 不修改工资 API、数据库、迁移、Worker、StatisticsModule 或工资统计口径。
- 不实现多个有效工资档案、取消到账、删除工资、官方账户查询、CSV 导出或工资通知。
- 不把工资数据放入情侣账本页面，不新增生产依赖或静态假数据。

## 决策记录

- `/salary` 并行加载档案、当年记录、余额和年度汇总；无档案时显示首次设置，而不展示伪造工资卡片。
- `/salary/:year/:month` 先按年度完整读取记录并匹配目标月份；不存在时从默认模板显式创建或选择复制紧邻上月，存在且未到账时允许替换项目，已到账时只读。
- 档案设置和月度项目复用可访问的编辑组件；模板允许 0 分，月度项目必须为正整数分，至少保留一个收入项目。
- 创建档案、创建月度记录和确认到账分别维护失败重试稳定幂等键；成功后才轮换，快速重复提交只发一次写请求。
- 年度趋势只在存在工资记录时渲染；始终消费服务端完整 12 个月数据，不从局部分页推导累计或缺月。
- 年度页面固定展示“数据来自工资记录，不代表官方账户余额”，专项名称使用批准的现代中文名称。

## 风险

- `/salary/year/:year` 必须排在 `/salary/:year/:month` 前，避免动态路由误吞。
- 列表 API 只有年份过滤，需要完整读取分页后再匹配月份；不得只读取第一页。
- 模板 0 分项目不能直接提交为月度项目；创建月度记录前必须过滤或要求补全为正金额。
- 已到账记录必须完全依赖服务端 `canEdit=false` 和 `paymentStatus=PAID` 呈现只读，到账操作需二次确认。
- 负余额、零剩余天数、空年度、长雇主/项目名和 320px 宽度需要独立状态与视觉测试。
