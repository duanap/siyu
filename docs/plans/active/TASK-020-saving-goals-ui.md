- 推理：Medium
- 是否使用子代理：否

## 目标

- 实现攒钱目标列表、新建和详情三条移动端路由，支持个人与有效朝暮同笺账本切换和刷新恢复。
- 展示服务端统一计算的目标金额、已存、剩余、整数万分比进度、目标日期、状态和成员贡献。
- 支持目标创建、OWNER 编辑/删除，以及本人存入记录新增、编辑和删除。
- 覆盖加载、空、错误、只读/无权限、长文本、提交失败、快速重复点击和日间/夜间状态。

## 非目标

- 不修改攒钱 API、数据库 Schema、迁移、服务端权限或汇总规则。
- 不将目标或存入同步为普通账目，不实现自动扣款、提现、预算、利息、封面上传或完成通知。
- 不修改工资、借贷、周期、统计、Worker、管理后台和 CSV 行为，不新增生产依赖。

## 风险

- 客户端若使用浮点或自行累计进度会与服务端口径漂移；展示只消费整数分和 `progressBasisPoints`。
- 快速点击或网络失败重试若重建幂等键会重复创建；目标和存入分别维护成功前稳定的幂等键，并用 busy 门禁。
- 情侣 MEMBER 与 OWNER 权限不同；目标和存入动作只根据服务端能力字段渲染，不在客户端复制角色规则。
- 详情操作后需要重新获取服务端汇总和状态；不得本地增量猜测已存金额。
- 列表分页若只取首页会遗漏目标；API 封装完整收集分页并设置上限防止无限循环。
