## 目标

- 实现受保护的 `/debts`、`/debts/new`、`/debts/:id` 手机端页面。
- 提供当前总负债、当前总待收款、本月已还、本月已收、即将到期和已逾期的客户端统计。
- 完成借贷创建、分页列表、详情、编辑、二次确认删除，以及多次还款/收款和可选同步普通账目。
- 直接使用 TASK-010 的 6 个借贷 API 和生成类型，覆盖私有边界、幂等重试、整数分和完整页面状态。

## 非目标

- 不实现借贷提醒 Worker、站内通知、后台管理或情侣共享借贷。
- 不修改普通账目来源维护、借贷删除策略或 TASK-012 以后模块。

## 关键设计

- 借贷始终为本人私有，不提供账本或情侣成员切换；客户端不缓存或共享借贷明细。
- 列表汇总只基于已加载的完整借贷集合；顺序拉取全部分页并设置 100 页安全上限，避免用首屏数据伪造统计。
- 本月已还/已收基于详情返回的处理记录计算，因此汇总加载会按当前有效借贷读取详情；任一详情失败时明确显示统计不可用，不展示不完整金额。
- 创建和处理草稿分别持有稳定 `debt-` / `debt-transaction-` 幂等键，失败重试保持，成功后轮换。
- 所有金额以字符串输入并通过既有 BigInt 工具转为安全整数分；超额处理在客户端预防，服务端冲突仍单独处理。
- 删除仅消费服务端 `canDelete`，使用二次确认；存在同步账目的借贷显示不可删除原因。

## 风险

- 列表 API 没有独立统计端点，若只统计当前页会产生错误总额。
- 本月处理统计需要详情记录，分页和详情并发需支持取消、失败完整性和请求数量上限。
- 同步账目依赖个人账本和启用的系统“其他”分类，失败不能误报处理成功。
- 创建/处理重试若轮换幂等键会重复写入；金额浮点转换会破坏财务准确性。
- 320px、长对方名称、软键盘和底部操作区可能导致溢出或遮挡。
