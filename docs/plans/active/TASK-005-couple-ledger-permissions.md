# TASK-005：情侣账本邀请、成员与权限

## 状态

已完成

## 模型与推理

- 推荐模型：Sol
- 推理：Extra High
- 是否使用子代理：否

## 目标

- 实现情侣账本创建、查询、改名、邀请、接受邀请、成员退出、所有权转移和解散。
- 所有写操作遵循事务、并发锁、幂等、审计和服务端权限边界。
- 实现手机端“朝暮同笺”邀请与成员管理页面，覆盖完整页面状态。

## 非目标

- 不实现分类 CRUD、普通账目、统计、通知、周期、工资、借贷或攒钱目标。
- 本任务只冻结情侣分类作用域规则；分类数据模型和页面由 TASK-006 实现。
- 不重新制作或部署 QQ 互联审核临时页面。

## 风险

- 并发接受邀请可能突破两人上限或让一个用户加入多个情侣账本，必须按用户和账本加事务级锁并在锁内复查。
- 所有权转移必须同时更新账本所有者和双方角色，任何失败不得留下部分状态。
- 明文邀请码只能在创建响应中返回一次，日志、数据库和审计内容不得保存。
- 解散只撤销访问，不能物理删除历史财务关系。
