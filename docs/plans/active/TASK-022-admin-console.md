# TASK-022：最小管理后台

## 状态

进行中

## 目标

- 交付管理端运行概览、用户状态、账本成员关系、周期执行状态与审计查询。
- 为用户停用/启用和失败周期实例人工重试提供独立权限、理由、确认与审计。
- 后台仅暴露运营所需的脱敏字段，不允许修改财务明细。

## 非目标

- 不实现公告、反馈、上传、异步导出或未批准的数据模型。
- 不读取或修改账目金额、工资、借贷、攒钱目标等敏感财务明细。
- 不新增生产依赖，不提供任意 SQL、角色分配或数据修复入口。

## 相关规则与验收

- BR-ADMIN-001 至 008、BR-AUDIT-001。
- AC-ADMIN-001 至 007、AC-RECUR-007、AC-QUALITY-001。
- ADR-008、ADR-013、ADR-031。

## 冻结口径

- 所有后台业务接口同时要求 `admin:access` 与对应最小权限；权限来自数据库 RBAC 和服务端 Guard。
- 用户列表只返回 ID、昵称、脱敏邮箱、状态、角色、创建时间；停用时撤销全部活跃会话，禁止操作者停用自己。
- 账本关系只返回账本类型/状态、成员角色/状态和脱敏身份，不返回账目或任何金额。
- 周期任务只返回执行状态、计划日、次数、安全错误码和规则摘要；仅 FAILED 可由具备权限的管理员同步人工重试。
- 审计列表不返回 IP 哈希或任意 before/after 明细；查询本身写审计。
- 所有列表按稳定字段倒序分页；后台写操作必须提供 2 至 200 字理由并写审计。

## 依赖与风险

- 停用用户必须与会话撤销同事务完成，否则已有 Access Token 仍可使用。
- 人工重试复用周期领域锁、数据库幂等和失败实例；不能直接篡改实例或账目。
- 审计查询会产生新的审计事实，需先取查询快照再写读取审计。
- 旧管理员需重新登录后，Access Token 才携带新增权限。
