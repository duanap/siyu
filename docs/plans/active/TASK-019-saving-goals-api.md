## 目标

- 实现个人与有效情侣账本的攒钱目标列表、创建、详情、更新和软删除 API。
- 实现本人存入记录的创建、更新和软删除，并在同一事务内精确重算目标已存金额与完成状态。
- 返回初始金额归创建者的成员贡献汇总、完整有效存入记录和服务端能力字段。
- 通过幂等事实、目标级锁、数据库约束和完整 E2E 保证重试及并发存入不重复、不丢失、不越权。

## 非目标

- 不实现 TASK-020 攒钱目标移动端页面。
- 不把存入记录同步为普通收入或支出，不实现自动扣款、提现、预算、利息或目标通知。
- 不修改工资、借贷、周期、统计、Worker、管理后台或 CSV 行为，不新增生产依赖。

## 决策记录

- 有效个人账本 OWNER 和情侣账本双方有效成员均可创建、读取目标和添加本人存入；目标更新/删除只允许账本
  OWNER，存入更新/删除只允许该记录本人。退出成员、非成员和已解散账本统一按资源不可见处理。
- 创建目标新增用户级幂等键与不可变规范请求哈希；创建存入沿用用户级幂等键并新增不可变请求哈希。同键同载荷
  返回原结果，同键异载荷冲突，原资源已软删除时不复活。
- 目标详情返回有效存入记录和成员贡献汇总；初始金额直接计入创建者汇总，不伪造存入记录。列表返回相同口径
  的目标摘要和贡献汇总，所有金额以整数分返回。
- 目标级 advisory transaction lock 串行化目标更新、存入增改删和汇总重算；每次从有效存入事实重新聚合，
  不使用读后增量加减。数据库延迟约束同时校验汇总和状态。
- `savedCent >= targetCent` 时状态为 `COMPLETED`，否则为 `ACTIVE`；修改目标金额、存入或删除存入均可双向切换。
  删除目标设置 `CANCELLED + deletedAt`，存入历史保留但不再可访问或修改。
- 目标和存入的归属、创建事实、金额/文本/日期边界与状态一致性由第十一迁移固化；写入时数据库再次要求操作者
  是有效账本成员，历史成员退出不级联删除既有事实。

## 风险

- 并发创建或修改存入可能导致汇总丢失更新；必须在同一目标锁内重算并用延迟约束兜底。
- 只按目标 ID 或存入 ID 裸查会泄露情侣关系和金额；Repository 查询必须同时包含有效账本成员边界。
- 贡献幂等键当前仅按用户全局唯一；新增请求哈希时需要为潜在历史记录生成稳定 legacy 值并禁止后续修改。
- 目标软删除后贡献事实仍存在；删除状态、汇总约束和删除策略必须允许保留历史且禁止继续写入。
