# 权限模型

## 角色

- 用户
- 情侣账本所有者
- 情侣账本成员
- 运营管理员
- 审计管理员

## 数据访问矩阵

| 数据 | 本人 | 情侣另一方 | 账本所有者 | 运营管理员 |
|---|---:|---:|---:|---:|
| 个人账本账目 | 读写 | 否 | 不适用 | 默认不看明细 |
| 情侣共同账目 | 读；自己的可写 | 读；自己的可写 | 全部管理 | 仅故障查询 |
| 个人账本分类 | 本人查看使用和全部管理 | 否 | 不适用 | 默认不看明细 |
| 情侣账本分类 | 双方查看使用；创建者管理自定义 | 同左 | 系统/自定义/排序全部管理 | 仅故障查询 |
| 个人借贷 | 读写 | 否 | 否 | 默认不看明细 |
| 工资/社保/医保/公积金 | 读写 | 否 | 否 | 默认不看明细 |
| 个人周期规则 | 读写 | 否 | 否 | 查询任务状态 |
| 情侣周期规则 | 双方可读；创建人可写 | 同左 | 全部管理 | 查询任务状态 |
| 个人攒钱目标 | 读写 | 否 | 否 | 默认不看明细 |
| 情侣攒钱目标 | 双方读写贡献 | 同左 | 目标管理 | 默认不看明细 |
| 审计日志 | 本人有限查看 | 否 | 否 | 按角色 |

情侣关系操作：双方均可查看账本和有效成员；只有所有者可改名、创建邀请、转移所有权和解散；
普通成员可退出，所有者必须先转移所有权或解散。退出、转移和解散后权限按事务提交结果立即失效。

分类操作：服务端列表返回 `canCreate/canReorder` 和每项 `canEdit/canToggle`。情侣普通成员可创建并管理
自己创建的自定义分类，但不可修改/启停系统分类、他人分类或执行全局排序；非成员、已退出成员和已解散
账本统一按资源不可见处理。客户端仅消费能力字段，不能推导或扩大权限。

## 后端校验顺序

1. 用户已认证且状态有效。
2. 资源未删除且状态允许操作。
3. 用户是目标账本有效成员或私有数据所有者。
4. 角色允许目标动作。
5. 业务状态允许动作。
6. 写入审计上下文。

## 防越权

禁止只按资源 ID 裸查。应在查询中包含授权条件，或统一使用 Policy/Repository。

## 管理后台

- 默认只查看汇总、状态和脱敏字段。
- 查看敏感详情需要专门权限、用途说明和审计。
- 后台不直接修改用户账目、工资和借贷金额。
- 修复数据通过审批后的迁移或运维脚本执行。
