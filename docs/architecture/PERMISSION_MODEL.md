# 权限模型

## 角色

- 用户
- 情侣账本所有者
- 情侣账本成员
- 运营管理员
- 审计管理员

## 数据访问矩阵

| 数据 | 本人 | 情侣另一方 | 账本所有者 | 运营管理员 |
|---|---:|---:|---:|---:|
| 个人账本账目 | 读写 | 否 | 不适用 | 默认不看明细 |
| 情侣共同账目 | 读；自己的可写 | 读；自己的可写 | 全部管理 | 仅故障查询 |
| 个人账本分类 | 本人查看使用和全部管理 | 否 | 不适用 | 默认不看明细 |
| 情侣账本分类 | 双方查看使用；创建者管理自定义 | 同左 | 系统/自定义/排序全部管理 | 仅故障查询 |
| 个人借贷 | 读写 | 否 | 否 | 默认不看明细 |
| 工资/社保/医保/公积金 | 读写 | 否 | 否 | 默认不看明细 |
| 个人周期规则 | 读写 | 否 | 否 | 查询任务状态 |
| 情侣周期规则 | 双方可读；创建人可写 | 同左 | 全部管理 | 查询任务状态 |
| 个人攒钱目标 | 读写 | 否 | 否 | 默认不看明细 |
| 情侣攒钱目标 | 双方读写贡献 | 同左 | 目标管理 | 默认不看明细 |
| 本人通知 | 读、标记已读 | 否 | 不适用 | 默认不看内容 |
| 审计日志 | 本人有限查看 | 否 | 否 | 按角色 |

情侣关系操作：双方均可查看账本和有效成员；只有所有者可改名、创建邀请、转移所有权和解散；
普通成员可退出，所有者必须先转移所有权或解散。退出、转移和解散后权限按事务提交结果立即失效。

分类操作：服务端列表返回 `canCreate/canReorder` 和每项 `canEdit/canToggle`。情侣普通成员可创建并管理
自己创建的自定义分类，但不可修改/启停系统分类、他人分类或执行全局排序；非成员、已退出成员和已解散
账本统一按资源不可见处理。客户端仅消费能力字段，不能推导或扩大权限。

账目操作：个人 OWNER 和情侣 OWNER 可管理账本内全部手工账目；情侣 MEMBER 可查看双方账目并创建，
但只能修改、删除自己创建的手工账目。`canEdit/canDelete` 只按当前操作者的状态、成员关系、角色、
记录创建归属和来源类型计算；创建人后来被禁用不影响有效 OWNER 的管理权限。非 MANUAL 来源账目只读。
软删除重试同样必须先完成账本可见性和防枚举检查，再验证当前操作者权限、来源和版本。

周期规则操作：个人和情侣有效成员可在所属账本创建规则；情侣双方可读，规则创建人可修改、暂停、恢复、
删除及处理自己的实例，账本 OWNER 可管理全部情侣规则和实例。服务端返回规则能力和实例
`canConfirm/canSkip`；普通成员不能扩大为可写对方创建的规则。退出成员、非成员和已解散账本统一按资源不可见处理。

工资操作：档案、月度记录、到账确认和同步生成的个人收入均只允许工资所属本人读取。确认到账要求本人账号
有效，记录查询固定 `userId`；情侣成员、其他用户和可猜测 ID 统一返回资源不可见。同步来源收入仍受个人账本
OWNER 成员关系约束，普通 Entry 接口只能读取本人可见的来源明细，不能修改或删除。

工资统计：年度汇总始终以当前认证用户的 `userId` 过滤工资记录和项目。工资余额先按当前用户选择工资记录，
再只查本人仍为有效 OWNER 的 PERSONAL 账本；不得接受客户端账本 ID，也不得聚合情侣账本、对方工资或对方
个人支出。没有本人到账记录时只返回空态，不泄露其他用户是否存在工资数据。

攒钱目标操作：个人目标只允许本人 PERSONAL 账本 OWNER 访问。情侣账本双方有效成员均可读取目标、创建目标
和添加本人存入；目标更新与删除只允许当前账本 OWNER，存入更新与删除只允许记录本人。服务端返回
`canManage/canContribute` 及存入 `canEdit/canDelete`，客户端不得自行扩大权限。退出成员、非成员、已解散账本、
软删除目标和目标/存入 ID 不匹配统一按资源不可见处理；历史创建人或贡献人身份不授予当前访问权。

通知操作：列表和未读数只能以当前认证用户 `userId` 过滤；批量已读必须在更新条件中同时限定本人和
`readAt IS NULL`。请求混入他人、不存在或已读 ID 时不返回逐项命中信息。通知的 `relatedType/relatedId`
不产生目标资源权限，跳转后的业务接口继续按当前成员关系、归属和资源状态独立授权。

## 后端校验顺序

1. 用户已认证；写操作还要求当前用户状态有效。
2. 资源未删除且状态允许操作。
3. 用户是目标账本有效成员或私有数据所有者。
4. 角色允许目标动作。
5. 业务状态允许动作。
6. 写入审计上下文。

## 防越权

禁止只按资源 ID 裸查。应在查询中包含授权条件，或统一使用 Policy/Repository。

## 管理后台

- 默认只查看汇总、状态和脱敏字段。
- 查看敏感详情需要专门权限、用途说明和审计。
- 后台不直接修改用户账目、工资和借贷金额。
- 修复数据通过审批后的迁移或运维脚本执行。
- `admin:access` 只允许进入管理端；概览、用户读取、用户状态写入、账本关系、任务读取、任务重试和审计读取
  分别要求对应权限，前端菜单不参与授权判断。
- 用户停用与会话撤销同事务完成，禁止当前管理员停用自己或最后一个活跃管理员；任务重试只接受 FAILED 周期实例并复用领域服务。
